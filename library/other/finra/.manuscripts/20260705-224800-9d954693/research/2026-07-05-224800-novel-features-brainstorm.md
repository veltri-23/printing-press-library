## Customer model

**Priya Patel — Compliance Ops Analyst at a mid-size broker-dealer.**
*Today:* Priya keeps three separate browser tabs open — the FINRA API Console for ad-hoc CRD lookups, a shared Postman collection hitting `registration/registrationValidationIndividual` for new-hire checks, and an Excel workbook where she pastes weekly CSV exports of the Reg SHO Threshold List and the firm's 4530 complaint filings so she can diff them by hand against last week's copy.
*Weekly ritual:* Monday morning "clean list" check on every newly registered rep's CRD status and disclosures; a midweek re-pull of the Threshold List to see whether any symbol the firm trades has newly appeared (five consecutive days of failure-to-deliver triggers a mandatory close-out); Friday's 4530 filing review, manually diffing this week's export against last week's to catch new complaints without opening every record.
*Frustration:* No tool treats "did this symbol newly cross the 5-day threshold escalation, and is it still there" as one answerable question — she diffs CSVs in Excel every week, and a missed day means a security could ratchet into mandatory close-out territory and she wouldn't know until an audit surfaces it later.

**Dan Okafor — Fixed Income Market-Data Researcher at an asset manager.**
*Today:* Dan has ad-hoc Python scripts in a shared repo that hit the Query API by hand for TRACE, Corporate Debt Market Breadth, Corporate Debt Market Sentiment, and Agency Debt Market Sentiment, pasting the results into a recurring internal markdown "market health" writeup.
*Weekly ritual:* Every Monday before the writeup, he re-pulls TRACE volume plus breadth/sentiment for the past week and manually joins them in a spreadsheet to compute week-over-week deltas, eyeballing whether sentiment swung meaningfully against breadth.
*Frustration:* Each dataset is a separate live pull with no local history — comparing this week's sentiment reading against four weeks ago means keeping his own shadow copies, since FINRA's API returns only current data per call and every third-party wrapper he's tried covers a single dataset with no cross-dataset join.

**Angela Reyes — Registration/Licensing Filing Coordinator at a broker-dealer's HR team.**
*Today:* Angela manually checks CRD status in the API console before submitting U4 amendments, and separately pulls Individual Delta / Branch Delta feeds to catch registration status changes (bar, suspension, termination) that HR needs to react to same-day.
*Weekly ritual:* Every week she pulls the delta feeds to see what changed since her last check, then cross-references CRDs against the firm's active-rep roster spreadsheet to flag anyone whose FINRA status diverged from HR's records — and separately, before submitting a batch of registration filings, she re-checks each rep's current registration validity one CRD at a time.
*Frustration:* No tool diffs "what changed in the delta feed since I last checked" against a locally held roster, and no tool lets her validate a batch of CRDs in one call — both are done by hand, CRD by CRD, in Excel.
