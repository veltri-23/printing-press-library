# Restore 8 Dropped Commands Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restore the 8 commands the PP 4.14.0 reprint dropped (`vibes`, `burn`, `budget`, `sessions`, `ship`, `tail`, `tree`, `custom-model`) as first-class commands coexisting with the new ones, so anyone on the 2026-05-15 build keeps their full command surface and output.

**Architecture:** Port the old command source + its shared helper file from the fork (`mvanhorn/printing-press-library`) into the current `suno-pp-cli` tree. The old helper file is already written against the current store API, so the port is mechanical: fix import paths, add `ctx` to client calls, drop 2 colliding helpers, normalize file headers. Read-commands keep reading the generic `resources` table (true-verbatim); `budget` gets its cap-enforcement re-wired into the new generate path. Add unit tests, update SKILL.md/README, and pass the printing-press ship gates.

**Tech Stack:** Go 1.x, cobra, SQLite (`internal/store`), the printing-press skill toolchain (verify-skill, scorecard, dogfood).
