# Codex Task 4 — GraphQL transport rewrite

## Scope (read-only API, no mutations)

You are rewriting the transport layer of a Go CLI generated by the printing-press scaffolder. The scaffolder emits REST (POST/GET/PUT/PATCH/DELETE) but the target API (Jobber) is **GraphQL-only**.

**Working directory:** `C:\Users\melan\printing-press\library\jobber\`

**Hard constraint:** This CLI is read-only. Do NOT add any code path that calls a Jobber mutation, regardless of what the scaffolder emitted. The only operations that ship are paginated reads (Relay connections) and id-lookups (clients, jobs).

## What to build

### 1. New file: `internal/client/graphql_ops.go`

A registry mapping fake REST paths to GraphQL operations. Required entries:

| Fake REST path        | GraphQL operation       | Connection root  | Notes |
|-----------------------|-------------------------|------------------|-------|
| `/clients`            | query, paginated        | `clients`        | filters: isCompany, isLead, isArchived, updatedAt, createdAt, tags |
| `/clients/{id}`       | query, single           | `client(id:..)`  | id-lookup |
| `/properties`         | query, paginated        | `properties`     | filters: clientId |
| `/quotes`             | query, paginated        | `quotes`         | filters: clientId, quoteStatus, createdAt, updatedAt |
| `/jobs`               | query, paginated        | `jobs`           | filters: clientId, jobStatus, createdAt (NOT updatedAt — Jobber doesn't support it) |
| `/jobs/{id}`          | query, single           | `job(id:..)`     | id-lookup |
| `/visits`             | query, paginated        | `visits`         | filters: jobId, completedAt (NOT updatedAt) |
| `/invoices`           | query, paginated        | `invoices`       | filter key is `clientId` (singular); also invoiceStatus, issuedDate |
| `/payment-records`    | query, paginated        | `paymentRecords` | returns `PaymentRecordInterface` — REQUIRES `__typename` + inline fragments; `PaymentRecordFilterAttributes.entryDate` is **exclusive both ends** — when callers pass a date range, pad ±1 day |

Each registry entry must expose:
- A `Query()` returning the GraphQL query string (use multi-line raw string literals; Relay `first`/`after`/`filter` variables required)
- A `Variables(params map[string]string) (map[string]any, error)` that converts the scaffolder's flat params bag into typed GraphQL variables. Skip empty strings (the generated commands always pass every flag, even unset ones, as empty strings — see `internal/cli/clients_list.go:36-46`).
- An `ExtractResponse(data json.RawMessage) (json.RawMessage, error)` that drills into `data.<connection>` and returns an envelope `{"nodes":[...], "pageInfo":{"hasNextPage":..., "endCursor":...}}`. For single-object queries, return the object directly.

### 2. New file: `internal/client/graphql_queries.go`

The actual GraphQL query strings. Keep them as Go string constants, one per operation. Field selections to start with (mirror the Python reference at `C:\Users\melan\Documents\Bouvier_Advisors\assignments\CRMs\Jobber\tenants\heritage_builders\probes\jobber_ar_recreation.py` lines 494-581 for invoice query shape):

- **Client fields:** `id name isArchived isCompany emails { description primary address } phones { description primary number } balance billingAddress { street city province postalCode country }`
- **Invoice fields:** `id invoiceNumber invoiceStatus issuedDate dueDate createdAt updatedAt receivedDate subtotal total invoiceNet syncStatus jobberWebUri client { id name balance } amounts { total subtotal discountAmount depositAmount paymentsTotal taxAmount nonTaxAmount invoiceBalance tipsTotal }`
- **Job fields:** `id jobNumber title jobStatus createdAt startAt endAt completedAt total client { id name } property { id street1 street2 city province postalCode }`
- **Visit fields:** `id title startAt endAt isComplete completedAt createdAt job { id jobNumber } client { id name }`
- **Quote fields:** `id quoteNumber title quoteStatus createdAt updatedAt amounts { total subtotal taxTotal depositAmount } client { id name } property { id street1 city }`
- **Property fields:** `id street1 street2 city province postalCode country client { id name }`
- **PaymentRecord fields:** Use `__typename` + inline fragments on at minimum `PaymentRecord` (entryDate amount paymentType adjustmentType jobberPaymentTransactionStatus client { id name } payments(first: 5) { nodes { __typename id invoice { id invoiceNumber } amount } })`. The interface forces fragment selection — a plain field list will fail.

For each Relay-connection query, the outer envelope MUST follow this canonical shape:

```graphql
query <Name>($first: Int!, $after: String, $filter: <FilterType>) {
  <connection>(first: $first, after: $after, filter: $filter) {
    pageInfo { hasNextPage endCursor }
    nodes { <field selection> }
  }
}
```

Use `has_field`-style hand-pruning only if you hit a schema error during dogfood — do NOT preemptively prune fields here.

### 3. New file: `internal/client/oauth.go`

OAuth refresh-token rotation with Windows User env persistence. This is **mandatory** — without it the CLI breaks within ~60 minutes.

Reference implementation: `jobber_ar_recreation.py` lines 193-231.

Exposed function: `func (c *Client) RefreshAccessToken() error`. Behavior:

1. POST to `https://api.getjobber.com/api/oauth/token` with JSON body:
   ```json
   {"client_id":"...","client_secret":"...","grant_type":"refresh_token","refresh_token":"..."}
   ```
   Pull the four values from `c.Config.ClientID`, `c.Config.ClientSecret`, `c.Config.RefreshToken`. Headers: `Content-Type: application/json`, `Accept: application/json`. **Do NOT send the Authorization header on this request.**
2. On success, parse `{"access_token", "refresh_token", "expires_in"}`. The refresh_token field MAY be omitted by Jobber on some refresh cycles — when omitted, keep the existing one.
3. Update `c.Config.AccessToken`, `c.Config.RefreshToken`, `c.Config.TokenExpiry` in memory.
4. Persist BOTH tokens to Windows User env via:
   ```go
   exec.Command("powershell", "-NoProfile", "-Command",
     `[Environment]::SetEnvironmentVariable('JOBBER_ACCESS_TOKEN', $env:JOBBER_ACCESS_TOKEN, 'User');` +
     `[Environment]::SetEnvironmentVariable('JOBBER_REFRESH_TOKEN', $env:JOBBER_REFRESH_TOKEN, 'User')`)
   ```
   The subprocess inherits the parent env, which already holds the new tokens because we set `os.Setenv("JOBBER_ACCESS_TOKEN", new)` and `os.Setenv("JOBBER_REFRESH_TOKEN", new)` first. Best-effort — log a warning on failure but don't return an error (the CLI works for this session even if persistence fails; the next session will use the not-yet-rotated refresh token from env, which Jobber accepts once).
5. **Redaction:** never log the access_token or refresh_token values. Log only first/last 4 chars at debug verbosity.

### 4. Modify `internal/client/client.go`

In `do()` (currently at line 266), before the dry-run branch, add a path-rewrite hook:

```go
if op, ok := lookupGraphQLOp(path); ok {
    return c.doGraphQL(op, method, params, body, headerOverrides)
}
```

Add a new method `doGraphQL` that:
1. Builds `{"query": op.Query(), "variables": op.Variables(params)}` JSON body
2. POSTs to `https://api.getjobber.com/api/graphql` (override targetURL — ignore `c.BaseURL` + path)
3. Reuses the existing retry/backoff/rate-limit/auth-header machinery
4. On HTTP 401 from a non-refresh request: call `RefreshAccessToken()`, retry the request ONCE with the new token. If the refresh itself fails, surface the error.
5. After successful response, runs `op.ExtractResponse(rawBody.data)` to unwrap the connection and returns the envelope. If `errors` is non-empty in the GraphQL response, return an `*APIError` with `StatusCode: 502` (synthetic — GraphQL APIs return 200 + errors-array on logical failures; we want classifyAPIError to treat this as a server-side problem).
6. **Throttle handling:** Jobber returns 200 OK with `errors[].extensions.code == "THROTTLED"` rather than HTTP 429. Detect this and feed it through the same `limiter.OnRateLimit()` + backoff path as a 429, using `time.Sleep(3 * attempt * time.Second)`.

- Do NOT touch `internal/cli/*.go` files in this task. A separate follow-up task will rename `jobber-jobs` -> `jobs` and flip `Hidden:true` to `false`.
- Do NOT touch the SQLite store layer.
- Do NOT touch the cache layer — GraphQL responses are cached the same as REST.
- Do NOT add a query-builder DSL or schema-introspection step. The query strings are hand-authored constants.
