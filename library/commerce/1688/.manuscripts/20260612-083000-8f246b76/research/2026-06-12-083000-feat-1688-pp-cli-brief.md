# 1688.com CLI Brief

## API Identity
- **Domain:** 1688.com — Alibaba's China-domestic B2B wholesale marketplace (the Mandarin-language, factory-direct counterpart to the English Alibaba.com). Products are 10-40% cheaper than Alibaba.com for the same SKU.
- **Users:** dropshippers, Amazon/eBay/Shopify sellers, sourcing agents, importers, procurement teams. They sit upstream of the supply chain choosing which Chinese factory/trader to buy from.
- **Data profile:** *offers* (product listings) carrying wholesale price (often tiered by quantity), MOQ, supplier/shop identity, factory-verification flags, **repurchase rate (回头率)**, 30-day transaction count, network-wide sales volume, region (province/city), product images, and service tags. *Suppliers* (shops) carry trade-service scores (composite/logistics/dispute/consultation) and verification badges.

## Reachability Risk
- **None — verified live this session (2026-06-12).**
- Search: `mtop.relationrecommend.WirelessRecommend.recommend` (`appId=32517`) on `h5api.m.1688.com` returned `ret:["SUCCESS::调用成功"]`, total `2000`, 20 offers — via the token-bootstrap → md5-sign → signed-call protocol over **plain HTTPS from a residential IP** (no Browserbase, no proxy, no API key).
- Detail: `https://detail.1688.com/offer/{offerId}.html` returned **HTTP 200, 590 KB real HTML, zero punish/block markers** from the same residential IP.
- **Caveat (documented, not blocking):** Browserbase data-center IPs ARE hard-blocked by Alibaba's `cloud_ip_bl` (terminal `action=deny`, no captcha) for `s./m./detail.1688.com`. The printed CLI replays via direct HTTPS from the user's own IP, so this does not apply. The mtop JSON host is on separate infra and is not punished.
- Probe-safe endpoint used: `GET mtop ... recommend` (read-only search).

## Top Workflows
1. **Keyword search → structured offers.** Search `手机壳`, get price / MOQ-tier / supplier / region / transaction-count / repurchase-rate per offer. (English keywords return sparse results; 1688's corpus is Mandarin — translate first.)
2. **Find the actual factory among resellers.** Many listings for the same item are traders reselling one plant. Combine `factoryInspection` + `superFactory` + `深度验厂` service tag + repurchase rate to surface real manufacturers.
3. **Compare price / MOQ / reliability across suppliers** for the same SKU before committing volume.
4. **Track price / repurchase / transaction drift over time** — re-sync a saved query and see which offers dropped price, which suppliers' reorder rates climbed.
5. **Supplier reliability profiling** — roll up trade-service scores, repurchase, and transaction volume per shop.

- Search-by-image (AliPrice extension) — **out of v1 scope**, documented as a known gap (needs image upload + a different mtop endpoint)

## Data Layer
- **Primary entities:** `offers` (offerId PK), `suppliers` (memberId/loginId PK)
- **Sync cursor:** (keyword, filters, page) — each search persists offers + supplier rows
- **FTS/search:** offline FTS on offer title + supplier name
- **Snapshots:** price_cny / repurchase_rate / booked_count per (offerId, synced_at) for drift tracking

## Why this CLI should exist
Every existing tool is either a **paid, stateless scraper API** or a closed extension. None: (a) run free on the anonymous mtop endpoint, (b) persist results to a local SQLite store, (c) track price/repurchase/transaction **drift over time**, or (d) compute sourcing-decision signals (factory-confidence, supplier reliability) locally. This CLI does all four, agent-native (`--json`/`--select`/`--csv`/typed exit codes), with no API key.
